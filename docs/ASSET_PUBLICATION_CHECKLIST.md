# Lista de verificación para publicar Proyecto Origen

## Antes de hacer público el repositorio

- [ ] Mantener una copia privada completa.
- [ ] Eliminar carpetas de Tiny Swords de la copia pública.
- [ ] Eliminar PNG, spritesheets y archivos `.aseprite`.
- [ ] Revisar archivos importados y cachés de Godot.
- [ ] Revisar Git LFS.
- [ ] Confirmar que los assets no estén en commits anteriores.
- [ ] Limpiar el historial si fueron subidos previamente.
- [ ] Agregar exclusiones al `.gitignore`.
- [ ] Mantener `THIRD_PARTY_ASSETS.md`.
- [ ] Mantener créditos a Pixel Frog.
- [ ] Probar la copia pública desde una ventana privada.
- [ ] Confirmar que el código y la documentación sean comprensibles sin distribuir assets.
- [ ] Verificar que capturas y videos no expongan archivos fuente descargables.
- [ ] No adjuntar packs ZIP ni enlaces directos a copias propias.

## Importante

`.gitignore` solamente evita que Git comience a seguir archivos nuevos. No elimina archivos que ya fueron registrados ni borra versiones anteriores del historial.
